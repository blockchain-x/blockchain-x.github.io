目前来讲，任何学习EOS相关技术的资料都来自https://github.com/EOSIO/eos
,因为eos开发更新的很快，网上教程在操作时候需要针对性修改才能通过。

先说下自己电脑的配置，我用的vmware虚拟机：

因为eos 脚本会检测内存少于8G就会报错，虽然有修改脚本办法绕过，但害怕后面总会暴露问题，所以老老实实的给分配了8.6G内存。

编译设置开发环境

获取EOS源码及其所有子模块

$ git clone https://github.com/eosio/eos --recursive
如果克隆代码时未带–recursive参数，那么可切换到项目路径下面执行如下命令，更新子模块。
$ git submodule update --init --recursive
上面要注意有时候某些eos的分支在编译的时候会出现mongodb 错误的问题，如果出现问题可以指定某些特定的分支，已知 dawn-v4.1.0 和 dawn-v3.0.0 是可以ubuntu编译的。

编译源码生成可执行文件

切换到上面的项目根目录下面，执行如下命令。
$ ./eosio_build.sh ubuntu full
期间如果遇到 Fail connect to github-production-release-asset-2e65be.s3.amazonaws.com 可以在/ets/hosts 加入一行 52.216.16.16 github-production-release-asset-2e65be.s3.amazonaws.com 来解决。
等待大概半小时到一小时编译好就会出现提示EOS.IO has been successfully built.

基于github 搭建 hexo 博客

发表于 2018-05-20

Hexo是基于NodeJs的静态博客框架，简单、轻量，其生成的静态网页可以托管在Github和Heroku上。

下面说下我在搭建这个博客过程中参考的东西和遇到的坑，首先整体是参考github链接：
Hexo博客搭建全攻略

过程中遇到的坑和需要注意点说下

node.js环境安装

这个需要安装最新版本，可以参考 Ubuntu16.04安装最新版nodejs （里面有详细的步骤跟安装过程中遇到问题的解决方式）

hexo安装速度十分慢

这个我曾经安装了一个多点小时都没进展，后来才知道因为国内环境所致，可以参考 hexo博客搭建时遇到的一些问题 这里面是用淘宝的npm分流——cnpm解决

关于主题的选择

网上大部分都是选择 hexo-theme-next 主题 ，不过现在主题 github 地址最新的已经不是 https://github.com/iissnan/hexo-theme-next ，而是迁移到了 https://github.com/theme-next/hexo-theme-next

github 本地的设置

按照文章 技术小白搭建个人博客 github+hexo 配置github ，并把 id_rsa.pub 文件设置到github的对应位置。
测试github是否能链接如果出现 ssh connect 错误可以参考 Agent admitted failure to sign

部署时候错误

hexo d 有时会出现 key 权限问题，有的人尝试在repositories 的settings设置key ，其实不用，（而且如果这里上传用户的key会提示Key is already in use而无法上传）。可以首先确保同一个shell下 ssh -T 能正确链接，然后删除 .deploy_git 文件夹，再次尝试。

访问慢的问题

因为博客主题使用google字体，加载很慢，需要解决参考文章
加速访问基于 Github Pages 和 Hexo 的博客 我是直接把Lato 字体禁止，也可以获得不错的速度和效果。
